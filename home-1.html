Clasificación Automatizada de Nódulos Pulmonares

Este proyecto de ingeniería aborda el desarrollo de un marco analítico integral para procesar tomografías computarizadas e identificar lesiones malignas, benignas o estados normales. A través de arquitecturas avanzadas y técnicas especializadas de visión artificial, el sistema mitiga el error interpretativo humano.

Validación experimental mediante k-folds con alta precisión.

Explorar Proyecto

¿Por qué es vital automatizar el diagnóstico?

Impacto y Mortalidad

El cáncer de pulmón es el tipo de cáncer más frecuente en ambos sexos en México. Cada año se registran más de 6,700 defunciones en el país debido a la alta agresividad de esta enfermedad.

El Factor Tiempo

El tratamiento efectivo depende de la detección temprana. Actualmente, solo el 0.6% de los tumores se diagnostican a tiempo, mientras que el 65% se detecta en periodos críticos.

Apoyo al Diagnóstico

La IA permite procesar tomografías con alta eficiencia. Estos sistemas apoyan al radiólogo reduciendo la variabilidad humana y minimizando falsos negativos críticos.

Distribución Analítica del Dataset

Uno de los desafíos más críticos en el entrenamiento de redes neuronales convolucionales aplicadas al entorno médico es el sesgo por desbalanceo de clases. Si un modelo se entrena con una abrumadora mayoría de casos malignos, desarrollará una preocupante tendencia a generar falsos positivos en entornos clínicos reales.

Para garantizar la equidad predictiva y un coeficiente Kappa de Cohen estadísticamente válido, se aplicó un pipeline automatizado de balanceo sintético y geométrico.

Impacto y Alcance del Proyecto

Estrategias de Preprocesamiento

Implementación de transformaciones complejas como Grid Distortion, CLAHE y Gauss Noise para simular variaciones clínicas reales. Esto permite que la IA aprenda bajo condiciones técnicas diversas.

Precisión de Alto Nivel

El modelo ResNet50 alcanzó una precisión del 99.82% y una sensibilidad del 99.81%. El sistema logra clasificar imágenes en un tiempo récord de solo 15.91 segundos.

Apoyo al Diagnóstico Médico

Generación de un marco reutilizable para el desarrollo de sistemas de detección temprana. El objetivo es reducir la brecha diagnóstica en México, donde solo el 0.6% de los casos se detectan a tiempo.

Metodología validada mediante esquema estratificado de k=5 folds.

Tecnologías y Metodología Aplicada

01

Arquitecturas CNN

Evaluación y adaptación de arquitecturas de aprendizaje profundo líderes en la industria: ResNet50, InceptionV3 y DenseNet201. Estas redes fueron entrenadas utilizando transfer learning para maximizar la extracción de características en imágenes médicas.

02

Preprocesamiento Avanzado

Implementación de técnicas críticas de procesamiento digital de imágenes como CLAHE para la mejora de contraste y Grid Distortion para simular variaciones clínicas reales. Esto permite que el sistema sea robusto ante diferentes condiciones de adquisición de tomografías.

03

Rigor Metodológico

Validación de resultados mediante un esquema de validación cruzada estratificada (k=5 folds) y un conjunto de prueba independiente. Este enfoque garantiza que la alta precisión obtenida sea reproducible y no producto de un sobreajuste de los datos.

04

Diagnóstico Multiclase

Clasificación automatizada de tomografías computarizadas en tres categorías esenciales para la toma de decisiones médicas: Benigno, Maligno y Normal (sano). El sistema busca reducir la variabilidad humana y detectar lesiones en etapas tempranas.

Comparativa de Modelos CNN

Este modelo demostró ser el más estable y eficiente computacionalmente bajo el escenario de validación. Alcanzó la mejor combinación de métricas con una precisión del 99.82%, una sensibilidad del 99.81% y un tiempo récord de clasificación de 15.91 segundos. Su arquitectura con conexiones residuales evitó el desvanecimiento del gradiente, consolidándolo como la opción óptima para el sistema de apoyo al diagnóstico propuesto.
ResNet50
El Modelo Ganador
Mostró un desempeño altamente competitivo durante la fase de entrenamiento y validación cruzada. Su arquitectura basada en módulos Inception permite extraer características a múltiples escalas de forma paralela, logrando una excelente precisión del 99.81%. Sin embargo, su complejidad estructural requirió un tiempo de clasificación mayor, registrando 24.46 segundos por lote de imágenes médicas.

Aunque su convergencia fue la más lenta del estudio con 25.84 segundos, demostró una notable capacidad para la identificación de características complejas en las tomografías.

Metodología de la Investigación

Flujo de trabajo detallado para la evaluación de modelos CNN

Fase 1: Adquisición y Análisis del Dataset

Se utilizó el dataset público The IQ-OTH/NCCD, el cual contenía originalmente 1,097 imágenes de tomografía computarizada. Se realizó un análisis inicial identificando un marcado desbalanceo entre las clases Benigno, Maligno y Normal, lo que motivó la necesidad de un tratamiento de datos especializado.

[ 1,097 Imágenes Originales ]

Fase 2: Preprocesamiento y Aumento de Datos

Para equilibrar el dataset y mejorar la robustez, se aplicaron técnicas avanzadas de Data Augmentation como Grid Distortion, CLAHE y Gauss Noise. Esto permitió expandir el conjunto a 3,000 imágenes totales (1,000 por clase), simulando variaciones clínicas y técnicas reales.

[ Balanceo: 3,000 Imágenes Totales ]

Fase 3: Selección de Arquitecturas y Transfer Learning

Se seleccionaron tres arquitecturas de aprendizaje profundo: ResNet50, InceptionV3 y DenseNet201. Se empleó la técnica de Transfer Learning utilizando pesos preentrenados en ImageNet, adaptando la capa de salida para la clasificación multiclase del problema.

[ Transfer Learning Aplicado ]

Fase 4: Entrenamiento con Validación Cruzada

Se implementó un esquema de Validación Cruzada Estratificada con k=5 folds. El dataset se dividió en una proporción de 90% para entrenamiento/validación y 10% para pruebas finales independientes, asegurando que el modelo generalizara correctamente sin sobreajuste.

[ Validación Cruzada k=5 ]

Fase 5: Evaluación de Desempeño y Métricas

La evaluación final consideró múltiples métricas: Precisión, Sensibilidad, F1-Score y Kappa de Cohen. Además, se analizó la complejidad temporal para determinar la eficiencia computacional de cada modelo en un entorno de diagnóstico real.

[ Evaluación Multivariable ]

Stack Tecnológico

Herramientas y librerías clave utilizadas en el desarrollo del sistema

Framework | Deep Learning

TensorFlow & Keras

Frameworks principales empleados para el diseño, compilación y entrenamiento de las arquitecturas neuronales convolucionales (ResNet50, InceptionV3, DenseNet201). Su flexibilidad permitió aplicar Transfer Learning de manera eficiente.

Librería | Visión Artificial

OpenCV

Herramienta fundamental para la manipulación y preprocesamiento de las tomografías computarizadas. Se utilizó para el redimensionamiento, filtrado y aplicación de algoritmos de mejora de contraste adaptativo (CLAHE).

Librería | Machine Learning

Scikit-Learn

Esencial para la validación y evaluación de los modelos. Facilitó la implementación de la Validación Cruzada Estratificada (Stratified K-Folds) y el cálculo de métricas de rendimiento como la Matriz de Confusión y el F1-Score.

Visualización | Análisis de Datos

Matplotlib & Seaborn

Librerías encargadas de generar las representaciones gráficas del comportamiento de la IA. Permitieron trazar las curvas de precisión/pérdida por cada época de entrenamiento y visualizar la distribución de las clases generadas.
